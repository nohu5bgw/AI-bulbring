The goal

Compilation engagements at MNP follow a tightly repeatable pattern: take the client’s raw GL and trial balance, map it to the chart of accounts, populate the CaseWare working paper file, generate the lead sheets, draft the financial statements, and surface anything that needs the partner’s eye. The mechanical work eats the bulk of the budget — the judgement is what actually matters.

The CaseWare Comp Agent is being built to handle the mechanical layer end-to-end — so an accountant spends their time on review and client questions, not data entry.

How it’s being designed to work
1

Ingest the raw file

Drop in the client’s trial balance, general ledger export, prior-year working papers, and any supporting schedules. The agent reads QuickBooks, Sage, Xero, and CSV/Excel exports.

2

Map to chart of accounts

AI matches each GL account to the MNP standard mapping — using prior-year mappings, account naming patterns, and balance behaviour. Anything ambiguous is flagged for review.

3

Populate working papers

Lead sheets, supporting schedules, adjusting entries, prior-year comparatives — auto-populated into the CaseWare file using MNP’s compilation templates.

4

Draft the financials

Balance sheet, income statement, cash flow, notes — drafted using the CSRS 4200 compilation reporting format and ready for partner review.

5

Flag what needs a human

Variance vs. prior year over a threshold? Unmapped accounts? Missing schedules? The agent calls it out in a single review checklist instead of burying it in the file.

6

Hand off to CaseWare

Output drops directly into the live CaseWare file structure — same templates, same naming conventions, same lead-sheet references. No re-keying.

Why this is worth building

Hours per file

Compilations should be a couple hours of review work, not days of data wrangling.

Consistency

Every file uses the same mapping logic — no more “how did last year’s engagement do it.”

CSRS 4200 ready

Targeting the standard compilation report format out of the box.

Local & private

Client data never leaves your machine — same model as the Bank Writeup Agent.

Prior-year aware

Reads last year’s file to keep mapping decisions consistent.

Review-first output

The agent shows its work — every entry has a trail back to source.
